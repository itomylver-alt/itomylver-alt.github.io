在已有的项目加Prometheus + Grafana 监控

##  技术架构
	docker
	 ├── nginx
	 ├── wordpress
	 ├── mysql
	 ├── redis
	 ├── prometheus      (采集指标)
	 ├── grafana         (展示监控)
	 └── cadvisor        (采集docker指标)
	数据流 
	 cadvisor ---> Prometheus ---> Grafana
     Prometheus 负责抓取数据
     Grafana 负责画图
  
##  配置
	在我们的docker-compose.yml 中加入
		    version: "3"

			services:

			  prometheus:
				image: prom/prometheus
				container_name: prometheus
				ports:
				  - "9090:9090"
				volumes:
				  - ./prometheus.yml:/etc/prometheus/prometheus.yml
				restart: always

			  grafana:
				image: grafana/grafana
				container_name: grafana
				ports:
				  - "3000:3000"
				restart: always
				volumes:
				  - grafana-data:/var/lib/grafana

			  cadvisor:
				image: gcr.io/cadvisor/cadvisor
				container_name: cadvisor
				ports:
				  - "8088:8080"
				volumes:
				  - /:/rootfs:ro
				  - /var/run:/var/run:rw
				  - /sys:/sys:ro
				  - /var/lib/docker:/var/lib/docker:ro
				restart: always

			volumes:
			  grafana-data:
			  
			   
    用yamllint docker-compose.yml 检查一下语法
    
     再创建prometheus.yml
       写入
       	global:
			  scrape_interval: 5s

		scrape_configs:
			  - job_name: "prometheus"
				static_configs:
				  - targets: ["localhost:9090"]

			  - job_name: "cadvisor"
				static_configs:
				  - targets: ["cadvisor:8080"]
	
	用yamllint 检查一下 yamllint prometheus.yml
	
	作用：Prometheus 每 5秒抓一次 cadvisor 指标


   然后运行 sudo docker-compose up -d 
               错误：我在docker-compose pull cAdvisor 的时候出错 
										cAdvisor 的官方镜像在 gcr.io（Google Container  Registry）
				而很多网络 访问 Google 很慢或直接失败 ，所以我改成了image: registry.cn-hangzhou.aliyuncs.com/google_containers/cadvisor:v0.47.0

   查看容器 sudo docker ps
