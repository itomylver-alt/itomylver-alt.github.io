## 描述
使用 Docker Compose 构建 WordPress + Redis + MySQL 架构，
实现 Web 与数据库分离部署。

使用 Nginx 作为反向代理服务器并实现负载均衡，
部署多个 WordPress 实例提高系统可用性。

通过 Redis 提供缓存加速，减少数据库压力。

## 添加Docker 
 ## 修改
 首先修改docker-compose.yml 文件
 	在service 下面添加
 		nginx:
           image: nginx
           container_name: nginx
        ports:
           - "80:80"
        volumes:
           - ./nginx.conf:/etc/nginx/conf.d/default.conf
        depends_on:
           - wordpress
然后创建nginx.conf   touch nginx.conf
		vim nginx.conf
			写入
				server{
						  listen 80;

							location / {

								proxy_pass http://wordpress:80;

								proxy_set_header Host $host;
								proxy_set_header X-Real-IP $remote_addr;

    }
}




## docker compose自定义网络

在docker-compose.yml里面加
networks:
   wpnet:
每个容器也要加
networks:
      - wpnet

写完后 用yamllint 检查一下语法

然后确认语法没问题
我们重启docker-compose : sudo docker-compose down              sudo docker-compose up -d

然后我们 sudo docker network ls  看到wordpress_wpnet 就可以了



##  部署wordpress 多实例 （负载均衡）

在docker-compose.yml 添加
	wordpress:
    image: wordpress
    container_name: wordpress
    restart: always

    ports:
      - "8888:80"

    environment:
      WORDPRESS_DB_HOST: 192.168.0.5:3307
      WORDPRESS_DB_USER: root
      WORDPRESS_DB_PASSWORD: 123456
      WORDPRESS_DB_NAME: wordpress
    depends_on:
      - redis
    networks:
      - wpnet

然后修改 nginx.conf
 改成

upstream wordpress_cluster{ 
        server wordpress:80;
        server wordpress2:80;
}

server{
  listen 80;

    location / {

        proxy_pass http://wordpress_cluster;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;

    }
}


 ##  MySQL 自动备份

     	在我们测试目录创建 
     mkdir mysql-backup
