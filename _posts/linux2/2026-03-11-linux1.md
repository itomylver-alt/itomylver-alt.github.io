搭建WordPress Redis Mysql Nginx 架构

涉及的技术：Docker ,Docker Compose ,Nginx, Mybatis ,Redis ,Mysql

我准备了两个Ubuntu的虚拟机   
	VM1 作为Web服务器 Docker , Nginx , WordPress, Redis
	VM2 作为数据库服务器  Mysql
	
	
	1首先 我们用 ip a 确定两台虚拟机的ip地址  VM1 是 192.168.0.4  VM2 是    192.168.0.5
	  
	
	2下载docker
	更新一下系统 sudo apt update
	下载docker sudo apt install docker.io -y

    启动docker sudo systemctl start docker           sudo systemctl enable docker
    
    然后docker -v 检查docker的版本
    
   3.在VM1 安装docker compose
   		sudo apt install docker-compose -y
        检查版本 docker-compose version
        
       在VM2 部署MySql 创建  mkdir mysql-docker  cd mysql-docker
       
       touch docker-compose.xml
       nano docker-compose.xml
       
       写入 
       	    version: '3'

			services:

			mysql:
				image: mysql:5.7
				container_name: mysql
				restart: always

				environment:
				MYSQL_ROOT_PASSWORD: 123456
				MYSQL_DATABASE: wordpress

				ports:
				 - "3307:3306"

				volumes:
				 - ./data:/var/lib/mysql
          
          启动 
        			docker-compose up -d
          查看
          			docker ps
          测试
          			docker exec -it mysql mysql -u root -p
          在VM1
          安装Mysql客户端
			sudo apt install mysql-client -y
			
			连接测试：
			mysql -h 192.168.0.5 -P 3307 -u root -p
			密码123456          
          
          4.在vm1 部署WordPress 和Redis
          mkdir wordpress-docker
          cd wordpress-docker
          
          创建docker-compose.yml
             编辑写入 nano docker-compose.xml
						version: "3.8"

						services:

						  redis:
							image: redis
							container_name: redis
							restart: always
							ports:
							  - "6379:6379"

						  wordpress:
							image: wordpress
							container_name: wordpress
							restart: always

							ports:
							  - "8888:80"

							environment:
							  WORDPRESS_DB_HOST: 192.168.0.5:3307
							  WORDPRESS_DB_USER: root
							  WORDPRESS_DB_PASSWORD: 123456
							  WORDPRESS_DB_NAME: wordpress

							depends_on:
							  - redis
		 	
		注意ip端口 数据库密码写对 
						
			
		启动
		
				我们可以用yamllint 来检查yml 的格式是否正确
				sudo apt install yamllint  下载
				然后yamllint docker-compose.yml 				 
				
				
				sudo docker-compose up -d
				
				在国内连不上docker 的镜像 所以配置一下国内的
					sudo mkdir -p /etc/docker
					sudo vim /etc/docker/daemon.json
						把这个写进去
								{
									  "registry-mirrors": [
										"https://docker.1panel.live",
										"https://hub.rat.dev",
										"https://docker.m.daocloud.io"
									  ]
									}				
			
			重启docker
			sudo systemctl daemon-reload
			sudo systemctl restart docker
			
			sudo docker info 			
			看里面是否有我们加的镜像
			
			测试一下
			sudo docker pull redis
			
			然后再执行
			sudo docker-compose up -d
			
			
				
		查看
				sudo docker ps   
				
				成功
				
		
		
				然后我们用浏览器访问 http://192.168.0.4:8888
				
				安装wordpress
				
				然后我们wordpress的插件安装 redis object cache
				
				点击启用
				
				redis 是为了 减少MySQL 查询  提升访问速度 提高并发能力
				
				因为我这里redis 端口冲突了 所以我们在vm1  docker里面没有编辑工具所以我们把配置文件复制下来
				
				docker cp wordpress://var/www/html/wp-config.php .
				然后编辑
				vim wp-config.php
				
				编辑添加
				define('WP_REDIS_HOST', 'redis');
                define('WP_REDIS_PORT', 6378);
				
				再复制回容器里面
				
				docker cp wp-config.php wordpress:/var/www/html/wp-config.php
				
				然后重新启动
				docker restart wordpress
				
				wordpress 还是无法访问 redis
				 看一下docker ps  有redis
				
				进入wordpress 容器 docker exec -it wordpress bash
				
				测试redis连接
				apt update
				apt install redis-tools -y
				
				redis-cli -h redis -p 6379
				
				如果返回PONG 说明网络正常
				
				是我的端口错了  同一台服务器的容器之间通信 要写容器名 容器端口  如果是跨服务器 那写服务器的映射端口
				
				现在redis 正常运行 我们 在wordpress 启用就好
